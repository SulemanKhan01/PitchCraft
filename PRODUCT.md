# Product

## Register

product

## Platform

web

## Users

Technical evaluators — developers and product managers assessing the AI/RAG pipeline quality. They're evaluating query betterment accuracy, retrieval precision, and generation quality. Context: benchmarking, demoing, or integrating PitchCraft's capabilities into their own systems.

## Product Purpose

PitchCraft bridges raw proposal PDFs and actionable proposal intelligence. Users upload past proposals, query them through a multi-stage RAG pipeline (spell correction, intent detection, rewriting, expansion, multi-query, keyword extraction), and generate tailored cover letters. Success means fast, accurate retrieval that demonstrates production-grade RAG — not a toy demo, but a system that handles real PDFs with real noise and returns precise, relevant answers.

## Positioning

Bridge between raw PDFs and actionable proposal intelligence — production-grade RAG that turns document chaos into queryable knowledge.

## Brand Personality

Fast-efficient. Gets out of your way. No hand-holding, no filler. The interface disappears so the pipeline's quality speaks for itself. Think: a well-documented API, not a SaaS landing page.

## Anti-references

Generic AI chat apps — minimal chat interfaces with no structure, no personality, and no indication of what's happening under the hood. PitchCraft should feel purposeful and transparent about its pipeline, not a black box with a text input.

## Design Principles

1. **Pipeline transparency** — Show, don't hide. The query betterment stages, retrieval confidence, and generation quality should be visible, not buried.
2. **Speed is the feature** — Every interaction should feel instant. Loading states, transitions, and feedback loops reinforce the "fast-efficient" personality.
3. **Tool, not toy** — This is a technical evaluation surface. No decorative elements, no unnecessary animation, no emoji-heavy empty states. Every pixel serves a purpose.

## Accessibility & Inclusion

No specific requirements at this stage. Basic keyboard navigation and screen reader support should be maintained as the foundation.
